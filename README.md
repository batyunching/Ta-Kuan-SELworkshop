# SEL 融入課程與對話研習網站需求確認稿

## 一、網站目的

本網站預計作為「SEL 融入課程以及對話」研習使用平台，協助參與教師完成以下任務：

1. 了解並申請 Google AI Studio 的 API Key。
2. 練習使用「導師暖心作戰手冊」網頁。
3. 練習使用「大觀課程溫度計：SEL 社會情緒學習 × 108 課綱素養指標評估系統」網頁。
4. 上傳練習成果，並在「成果發表區」集中瀏覽、放大投影與分享。

網站定位不是取代既有的兩個工具網站，而是作為研習現場的「入口、任務引導、成果收集與發表平台」。

## 二、使用者與登入方式

### 登入頁面

進入網站後，先顯示登入頁面，欄位包含：

- 身分選擇：「訪客」或「教職員」
- 姓名輸入

### 登入目的

本階段建議採用簡易登入，不需要密碼，目的如下：

- 讓成果上傳時能自動標記上傳者姓名。
- 方便成果發表區依姓名辨識作品。
- 讓系統可依登入資料找回該老師先前儲存的草稿與成果。
- 研習現場可快速進入，不增加帳號管理負擔。

一般老師登入仍採簡易登入；管理者後台另設密碼登入。若後續需要更嚴謹的管理，可以再新增「教職員驗證碼」或正式帳號驗證。

因為本網站需要支援「中斷後再次登入仍可修改資料」，建議後續設計時保留一個避免同名混淆的方式，例如「研習場次代碼」或「簡易識別碼」。若研習人數不多，也可以先用身分加姓名作為辨識依據。

## 三、登入後主頁結構

登入後進入研習主頁，建議分成四個主要區塊：

1. Google AI Studio API Key 的產生
2. 導師暖心作戰手冊網頁的使用
3. 大觀課程溫度計網頁的使用
4. 成果發表區

主頁可以採用「左側導覽列＋右側內容區」或「上方分頁導覽」方式呈現。研習現場若需要投影操作，建議使用上方分頁或大型按鈕式導覽，讓畫面更清楚。

## 七、區塊 4：成果發表區

### 主要目的

成果發表區用來集中呈現老師上傳的練習成果，方便研習講師在會議中瀏覽、切換、放大投影，或邀請老師上台分享。

### 成果分類

成果發表區需提供分類篩選，避免兩種成果混淆。

篩選項目建議：

- 全部成果
- 導師暖心作戰手冊
- 大觀課程溫度計

### 成果卡片內容

每一筆成果建議顯示：

- 成果類型
- 成果標題
- 上傳者姓名
- 簡短說明或心得
- 上傳時間
- 檔案縮圖或檔案圖示
- 「查看」或「放大」按鈕

### 放大投影功能

成果發表區需要具備適合投影的放大模式。

建議功能：

- 點選單一成果後進入全螢幕或大畫面檢視。
- 圖片成果可放大顯示。
- PDF 或文件成果可開啟預覽或下載。
- 顯示上傳者姓名、成果標題與成果類型。
- 提供上一件、下一件切換。
- 提供返回成果列表。

### 投影模式建議

投影模式建議採用簡潔版面：

- 字體放大。
- 背景乾淨。
- 減少不必要按鈕。
- 主要成果內容置中。
- 適合在教室、會議室或大螢幕觀看。

## 八、管理者後台

### 主要目的

管理者後台提供研習主持人或管理者整理成果資料，避免測試資料、重複上傳或不適合公開投影的成果留在發表區。

### 管理者登入

管理者後台需要輸入密碼才能進入。

- 管理者密碼：22725015
- 密碼正確後才能進入管理者後台。
- 一般訪客與教職員登入後不應看見刪除功能。

### 管理者後台功能

管理者後台建議提供以下功能：

- 查看所有老師上傳的成果。
- 依成果類型篩選：
  - 導師暖心作戰手冊
  - 大觀課程溫度計
- 依上傳者姓名搜尋。
- 依儲存狀態篩選：
  - 草稿
  - 已發表
- 單獨刪除某一筆成果。
- 批次選取多筆成果後刪除。

### 刪除確認機制

為避免誤刪，所有刪除操作都需要再次確認。

單獨刪除流程建議如下：

1. 管理者點選單一成果的「刪除」。
2. 系統顯示確認視窗，列出成果標題、上傳者姓名與成果類型。
3. 系統詢問管理者是否確定刪除。
4. 管理者按下「確認刪除」後，才執行刪除。
5. 管理者按下「取消」時，不刪除任何資料。

批次刪除流程建議如下：

1. 管理者勾選多筆成果。
2. 管理者點選「批次刪除」。
3. 系統顯示確認視窗，列出即將刪除的成果數量與成果清單摘要。
4. 系統再次詢問管理者是否確定批次刪除。
5. 管理者按下「確認批次刪除」後，才執行刪除。
6. 管理者按下「取消」時，不刪除任何資料。

### 刪除資料範圍

刪除成果時，需同步處理：

- Supabase Database 中的成果文字資料。
- Supabase Storage 中對應的上傳檔案。
- 成果發表區中的顯示資料。

若後續希望降低誤刪風險，也可以設計成先「移至回收區」，再由管理者進行永久刪除。

## 九、建議使用流程

1. 老師進入研習平台。
2. 選擇身分並輸入姓名。
3. 閱讀 API Key 申請說明並觀看影片。
4. 開啟或掃描「導師暖心作戰手冊」進行練習。
5. 回到研習平台，上傳第一項成果。
6. 開啟或掃描「大觀課程溫度計」進行練習。
7. 回到研習平台，上傳第二項成果。
8. 講師進入「成果發表區」。
9. 依成果類型篩選作品。
10. 使用放大投影功能進行分享。

## 十、資料儲存與續編機制

### 儲存平台

老師上傳的資料建議使用 Supabase 作為雲端資料儲存區。

Supabase 可分成兩個主要用途：

- Supabase Database：儲存成果的文字資料與狀態。
- Supabase Storage：儲存老師上傳的圖片或文件檔案。

### 建議儲存內容

每一筆成果建議保存以下資料：

- 使用者身分：訪客或教職員
- 使用者姓名
- 成果類型：導師暖心作戰手冊或大觀課程溫度計
- 成果標題
- 簡短說明或心得
- 上傳檔案位置
- 儲存狀態：草稿或已發表
- 建立時間
- 最後更新時間

### 中斷後續編需求

為避免研習現場因為網頁關閉、瀏覽器重新整理、電腦當機或網路暫時中斷導致資料遺失，網站需要支援續編機制。

建議行為如下：

- 老師填寫成果時，可先儲存為草稿。
- 老師上傳檔案後，檔案會存入 Supabase Storage。
- 文字資料與檔案位置會存入 Supabase Database。
- 老師再次登入後，系統會依身分與姓名載入先前儲存的草稿與成果。
- 老師可以繼續修改成果標題、說明或心得。
- 老師可以重新上傳檔案，或替換先前的檔案。
- 成果發表區預設顯示「已發表」成果，避免尚未完成的草稿被投影出來。

### 儲存操作建議

建議在成果上傳區提供以下按鈕：

- 儲存草稿
- 發表成果
- 更新成果
- 重新上傳檔案

系統也可以加入自動暫存功能，例如老師輸入文字後每隔一段時間自動儲存一次。不過第一版可先採用明確的「儲存草稿」按鈕，讓操作行為更容易理解。

## 十一、資料與隱私原則

建議遵守以下原則：

- 不收集老師的 Google AI Studio API Key。
- 上傳成果需標示姓名，方便研習發表使用。
- 若成果中含有學生個資、班級敏感事件或可識別學生資訊，需提醒老師先去識別化。
- 成果發表區主要用於研習現場分享，是否公開於網路需另行確認。
- Supabase 權限需設定妥當，避免未授權使用者任意查看、修改或刪除成果。
- 若使用簡易登入，成果資料仍建議以研習用途為主，不存放敏感個資。
- 管理者後台需限制一般使用者進入，刪除功能只能由輸入正確管理者密碼的人使用。

## 十二、後續需要確認的事項

正式進入網頁設計前，建議確認以下問題：

1. 網站是否只在研習當天使用，或之後也會長期開放？
2. Supabase 專案要使用哪一個帳號建立與管理？
3. 上傳檔案大小是否需要限制？
4. 成果發表區是否需要除了刪除之外，再新增「隱藏成果」功能？
5. 「訪客」是否也可以上傳成果，或只能瀏覽？
6. 是否需要匯出成果清單，例如 Excel 或 CSV？
7. 是否確認研習網頁正式顯示名稱使用「導師暖心作戰手冊」？
8. 是否需要加入研習場次代碼或簡易識別碼，以避免同名老師資料混淆？
